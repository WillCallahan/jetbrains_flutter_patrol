Develop integration tests with Hot Restart.

Usage: patrol develop [arguments]
-h, --help                                        Print this usage information.
-t, --target=<patrol_test/app_test.dart>          Integration test target to use as entrypoint.
--exclude=<patrol_test/flaky_test.dart>       Integration test targets to exclude.
--[no-]generate-bundle                        Whether to generate a bundled Dart test file.
(defaults to on)
-d, --device=<all, emulator-5554, 'iPhone 14'>    Devices to run the tests on. If empty, the first device is used.
--[no-]debug                                  Build a debug version of your app (default mode)
--[no-]profile                                Build a version of your app for performance profiling.
--[no-]release                                Build a release version of your app
--flavor                                      Flavor of the app to run.
--dart-define=<KEY=VALUE>                     Additional key-value pairs that will be available to the app under test.
--dart-define-from-file=<config/test.json>    Environment configuration from a provided path that will be availableto the app under test.
--[no-]label                                  Display the label over the application under test.
(defaults to on)
--test-server-port                            Port to use for server running in the test instrumentation app.
(defaults to "8081")
--app-server-port                             Port to use for server running in the app under test.
(defaults to "8082")
--tags=<(chrome || firefox) && !slow>         Tags to filter the tests by.
--[no-]hide-test-steps                        Hide test steps while running the tests.
--[no-]clear-test-steps                       Clear test steps after the test finishes.
(defaults to on)
--[no-]check-compatibility                    Verify if the dependencies are compatible between each other.
(defaults to on)
--[no-]uninstall                              Uninstall the app before and after the test finishes.
(defaults to on)
--build-name=<1.2.3>                          Version name of the app.
--build-number=<123>                          Version code of the app.
--package-name=<pl.leancode.awesomeapp>       Package name of the Android app under test.
--bundle-id=<pl.leancode.AwesomeApp>          Bundle identifier of the iOS app under test.
--clear-permissions                           Clear permissions available through XCUIProtectedResource API before running each test.
--full-isolation                              (Experimental) Uninstall the app between test runs on iOS Simulator to achieve full isolation.
--ios=<17.5>                                  Pass iOS version. If empty, `latest` will be used. This flag only works with iOS simulator.
--[no-]open-devtools                          Automatically open Patrol extension in DevTools when ready.

Run "patrol help" to see global options.
Read detailed docs at https://patrol.leancode.co/cli-commands/develop
